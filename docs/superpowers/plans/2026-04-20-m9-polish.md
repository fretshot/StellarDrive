**Files:**
- Modify: `app/api/chat/route.ts`

No new files. The check uses the existing `supabase` client (user-scoped, RLS-filtered) to count recent messages.

**Limit:** 20 user messages per 60 seconds. Returns HTTP 429 with `Retry-After` header.

- [ ] **Step 1: Add rate-limit check to chat route**

Open `app/api/chat/route.ts`. After the `sessionId` ownership check (around line 40) and before the "Extract last user message" block, insert:

```ts
// Rate limit: max 20 user messages per 60 seconds
const since = new Date(Date.now() - 60_000).toISOString();
const { count } = await supabase
  .from("chat_messages")
  .select("id", { count: "exact", head: true })
  .eq("role", "user")
  .gte("created_at", since);
if ((count ?? 0) >= 20) {
  return new Response(
    JSON.stringify({ error: "rate_limited", retryAfter: 60 }),
    { status: 429, headers: { "Content-Type": "application/json", "Retry-After": "60" } },
  );
}
```

Replace the select and return with:

    .from("action_previews")

      session_id: ctx.sessionId,
      message_id: ctx.messageId,
      org_id: ctx.orgId,
